Nombre Tipo (GoF) Descripción
Object Pool Patrones creacionales Se obtienen objetos nuevos a través de la clonación
Abstract Factory Patrones creacionales Nos provee una interfaz que delega la creación de un conjunto de objetos relacionados sin necesidad de especificar en ningún momento cuáles son las implementaciones concretas.
Builder Patrones creacionales Abstrae el proceso de creación de un objeto complejo
Factory Method Patrones creacionales Expone un método de creación, delegando en las subclases la implementación de este método.
Prototype Patrones creacionales Permite la creación de objetos basados en «plantillas». Un nuevo objeto se crea a partir de la clonación de otro objeto.
Singleton Patrones creacionales Restringe la instanciación de una clase o valor de un tipo a un solo objeto.
Model-View-Controler (MVC) Patrones creacionales Modelo Vista Controlador
Hierarchical MVC Patrones creacionales
Model-View-Adapter (MVA) Patrones creacionales
Model-View-Presenter (MVP) Patrones creacionales
Model-View View-Model (MVVM) Patrones creacionales
Adapter / Wrapper Patrones estructurales Permite a dos clases con diferentes interfaces trabajar entre ellas, a través de un objeto intermedio con el que se comunican e interactúan.
Bridge Patrones estructurales Desacopla una abstracción de su implementación, para que las dos puedan evolucionar de forma independiente.
Composite Patrones estructurales Facilita la creación de estructuras de objetos en árbol, donde todos los elementos emplean una misma interfaz.
Decorator Patrones estructurales Permite añadir funcionalidad extra a un objeto (de forma dinámica o estática) sin modificar el comportamiento del resto de objetos del mismo tipo.
Facade Patrones estructurales Es un objeto que crea una interfaz simplificada para tratar con otra parte del código más compleja, de tal forma que simplifica y aísla su uso.
Flyweight Patrones estructurales Una gran cantidad de objetos comparte un mismo objeto con propiedades comunes con el fin de ahorrar memoria.
Proxy Patrones estructurales Es una clase que funciona como interfaz hacia cualquier otra cosa: una conexión a Internet, un archivo en disco o cualquier otro recurso que sea costoso o imposible de duplicar.
Module Patrones estructurales Agrupa varios elementos relacionados, como clases, singletons, y métodos, utilizados globalmente, en una entidad única.
Chain of Responsibility Patrones de comportamiento Se evita acoplar al emisor y receptor de una petición dando la posibilidad a varios receptores de consumirlo. Cada uno tiene la opción de consumir esa petición o pasárselo al siguiente dentro de la cadena.
Command Patrones de comportamiento Son objetos que encapsulan una acción y los parámetros que necesitan para ejecutarse.
Interpreter Patrones de comportamiento Define una representación para una gramática así como el mecanismo para evaluarla.
Iterator Patrones de comportamiento Se utiliza para poder movernos por los elementos de un conjunto de forma secuencial sin necesidad de exponer su implementación específica.
Mediator Patrones de comportamiento Objeto que encapsula cómo otro conjunto de objetos interactúan y se comunican entre sí.
Memento Patrones de comportamiento Permite volver a estados anteriores del sistema.
Observer Patrones de comportamiento Los objetos son capaces de suscribirse a una serie de eventos que otro objetivo va a emitir, y serán avisados cuando esto ocurra.
State Patrones de comportamiento Permite que un objeto modifique su comportamiento cada vez que cambie su estado interno.
Strategy Patrones de comportamiento Permite disponer de varios métodos para resolver un problema y elegir cuál utilizar en tiempo de ejecución.
Template Method Patrones de comportamiento Especifica el esqueleto de un algoritmo, permitiendo a las subclases definir cómo implementan el comportamiento real.
Visitor Patrones de comportamiento Permite separar el algoritmo de la estructura de datos que se utilizará para ejecutarlo. De esta forma se pueden añadir nuevas operaciones a estas estructuras sin necesidad de modificarlas.
